国内主流自媒体平台发布规则查询与审稿 Skill

让 Codex 在做内容前先懂平台规则

这个交付包把抖音、小红书、微信视频号、微信公众号、B站、快手、微博的发布规格、素材建议、内容红线、AIGC 标识、版权和商业合规检查，整理成带 BID 前缀的可安装 Skill。它不是简单的尺寸表，而是一组可被 Codex 调用的“发布前检查员”：做封面、写标题、改图文、导出视频、审客户素材前，先让它按目标平台筛一遍风险。

交付内容

根目录是总交付包，每个平台放在自己的资产文件夹中。资产文件夹里包含真正可安装的 Skill 文件夹，以及同名的独立压缩包。安装时要复制“里面那层带 SKILL.md 的文件夹”，不要把整个资产文件夹直接放进 Skills 目录。

安装前准备

如果只是复制 Skill 文件夹，不需要安装复杂依赖。只有在使用抖音媒体检查脚本时，才需要 Python；如果要读取视频时长和编码，建议安装 ffprobe。

基础环境

Windows 本地 Codex 客户端或能读取 %USERPROFILE%\.codex\skills 的 Codex 环境。

必须本地 Skills 目录

Python

仅用于运行抖音媒体检查脚本。没有 Python 也不影响 Skill 的规则查询和审稿功能。

可选脚本检查

GitHub 包

公开仓库可作为云端备份或给客户下载：rjl2024/bid-self-media-platform-rules-skills。

可选远程下载

确认本机 Skills 目录

$dstRoot = "$env:USERPROFILE\.codex\skills"
New-Item -ItemType Directory -Force -Path $dstRoot | Out-Null

确认 Python 是否可用

安装步骤

下面给三种安装方式。最推荐“方式 A：一次安装全部平台”，适合客户拿到完整包后直接使用。

方式 A：一次安装全部平台 Skill

  1. 确认交付包位置

    本地交付包默认路径如下。如果你把文件夹移动到了别处，只需要改第一行 $srcRoot。

    $srcRoot = "I:\闲鱼发布产品库\063005自媒体平台规则skill发布包"
  2. 复制全部可安装 Skill

    这段命令会自动找到每个平台资产文件夹里真正带 SKILL.md 的子目录，并复制到本机 Skills 目录。如果本机已有同名 Skill，会先备份旧版本。

    $srcRoot = "I:\闲鱼发布产品库\063005自媒体平台规则skill发布包"
$dstRoot = "$env:USERPROFILE\.codex\skills"

New-Item -ItemType Directory -Force -Path $dstRoot | Out-Null

Get-ChildItem -Directory $srcRoot | ForEach-Object {
  Get-ChildItem -Directory $_.FullName | Where-Object {
    Test-Path (Join-Path $_.FullName "SKILL.md")
  } | ForEach-Object {
    $dst = Join-Path $dstRoot $_.Name
    if (Test-Path $dst) {
      Rename-Item $dst "$($_.Name)-backup-$(Get-Date -Format yyyyMMddHHmmss)"
    }
    Copy-Item $_.FullName $dst -Recurse
  }
}
  3. 重启 Codex 客户端

    复制完成后关闭并重新打开 Codex，让它重新扫描 Skills 目录。重启后即可在对话里点名调用，例如 $bid-douyin-platform-rules。

方式 B：只安装某一个平台

如果客户只需要一个平台，比如只需要小红书规则，可以只复制对应 Skill。

$src = "I:\闲鱼发布产品库\063005自媒体平台规则skill发布包\02-bid-xiaohongshu-platform-rules\bid-xiaohongshu-platform-rules"
$dst = "$env:USERPROFILE\.codex\skills\bid-xiaohongshu-platform-rules"

if (Test-Path $dst) {
  Rename-Item $dst "bid-xiaohongshu-platform-rules-backup-$(Get-Date -Format yyyyMMddHHmmss)"
}
Copy-Item $src $dst -Recurse

方式 C：从 GitHub 下载后安装

如果使用公开仓库版本，先下载或克隆仓库，再按方式 A 复制 Skill 文件夹。

git clone https://github.com/rjl2024/bid-self-media-platform-rules-skills.git
cd bid-self-media-platform-rules-skills

如果客户不熟悉 Git，也可以打开仓库页面下载 zip，解压后把里面的 Skill 子目录复制到 %USERPROFILE%\.codex\skills。

常见安装错误 不要把 01-bid-douyin-platform-rules 这一层复制到 Skills 目录后就结束。真正的 Skill 是它里面的 bid-douyin-platform-rules。正确结构应是 C:\Users\你的用户名\.codex\skills\bid-douyin-platform-rules\SKILL.md。

安装后验证

安装后先做一个不涉及真实发布的安全测试。测试目标不是让 Codex 生成内容，而是确认 Skill 能被识别、能读取 reference、能按平台输出规则和风险项。

只输出检查清单，不要生成具体内容。

标准工作流

建议把这些 Skill 用在“发布前”和“交付前”，而不是发布后补救。最稳的流程是先定平台，再做素材规格，再审内容风险，最后才进入真实后台发布。

  1. 确定目标平台和发布形态

    先明确是抖音短视频、小红书图文、公众号文章、B站视频，还是快手直播电商。不同形态的尺寸、风险、标题策略完全不同。

  2. 让 Codex 调用对应 Skill 给出规格

    让 Codex 输出推荐尺寸、硬性限制、需要后台复核的字段。不要只问“尺寸是多少”，要让它给出“发布前检查清单”。

  3. 制作或调整素材

    按推荐尺寸导出视频、封面、图文长图。涉及多平台分发时，不要用一套图硬发所有平台，至少为抖音、小红书、公众号、B站各准备适合的封面比例。

  4. 做合规审稿

    让 Codex 检查标题党、虚假夸大、广告未披露、站外导流、版权、AIGC、敏感垂类。输出时分成“必须改”“建议改”“后台复核”。

  5. 进入平台后台复核

    对大小、时长、张数、封面裁切、直播封面、商品链接、长文入口等账号相关字段，以创作者后台实时提示为准。

平台逐项说明

抖音：短视频、图文、直播、AIGC

$bid-douyin-platform-rules

适合在做短视频导出、抖音图文、直播封面、AIGC 视频、矩阵分发前调用。抖音 Skill 对“发布路径”特别敏感：App 手动发布、创作服务平台、OpenAPI、OpenSDK/H5 分享的限制并不完全一样。

推荐素材竖屏视频 1080x1920；横屏视频 1920x1080；图文封面 1080x1440、1080x1920 或 1080x1080。
重点检查其他平台水印、站外导流、二维码、搬运痕迹、未授权音乐/素材、虚假效果、AIGC 未标识。
后台复核视频大小与时长、图文张数、长文入口、直播封面比例、电商/生活服务挂载权限。
请使用 $bid-douyin-platform-rules，审查这个抖音短视频发布方案。
内容类型：知识口播视频
素材：1080x1920 竖屏视频、封面一张、标题 3 个备选
请检查：视频规格、封面安全区、标题风险、水印、导流、AIGC、版权，并输出必须修改项。

小红书：笔记、图文、种草合规

$bid-xiaohongshu-platform-rules

适合做小红书封面、图文笔记、种草文案、教程型笔记、合集型笔记。小红书更关注“真实体验”和“营销披露”，标题和正文不能只追求强转化。

推荐素材封面 1080x1440 或 1080x1350；方图 1080x1080；视频 1080x1920。
重点检查虚假种草、广告未披露、功效夸大、前后对比误导、私信/微信导流、医疗美容和健康产品宣称。
后台复核图片张数、单图大小、视频大小与时长、账号是否有商业合作或店铺能力。
请使用 $bid-xiaohongshu-platform-rules，把这篇内容改成小红书笔记。
要求：给 5 个标题、1 个封面文案、正文结构、标签建议。
另外标出种草、广告、功效宣称、私域导流风险。

微信视频号：视频、直播、微信生态链接

$bid-wechat-channels-platform-rules

适合做视频号短视频、直播预告、直播封面、商品或小程序挂载内容。视频号处于微信生态内，尤其要注意外链、私域、商品链接和直播场景的边界。

推荐素材竖屏视频 1080x1920；横屏视频 1920x1080；封面优先做中心安全区。
重点检查外链导流、私下交易、商品效果夸大、直播口播风险、未授权素材、AIGC 未说明。
后台复核视频号助手里的大小、时长、封面裁切、直播封面、商品/小程序挂载权限。
请使用 $bid-wechat-channels-platform-rules，检查这个视频号直播预告。
请重点看：封面比例、标题、直播主题、商品链接、导流话术、AIGC 和版权风险。

微信公众号：文章、封面、转载、素材

$bid-wechat-official-account-rules

适合公众号文章发布前审稿、封面图建议、标题党检查、素材版权检查、转载授权检查。公众号文章要特别关注外链、二维码、转载授权和敏感行业表述。

推荐素材主封面常用 900x383，同时准备方形裁切；正文图片宽度建议 900-1080 像素。
重点检查标题党、转载授权、图片版权、二维码外链、医疗金融教育表述、虚假数据和专家背书。
后台复核标题长度、摘要长度、封面裁切、外链权限、原创/转载/付费内容能力。
请使用 $bid-wechat-official-account-rules，审查这篇公众号文章。
请分成：标题风险、封面风险、正文合规风险、引用/转载/图片版权风险、发布前后台复核项。

B站：视频、封面、标题、标签

$bid-bilibili-platform-rules

适合教程视频、知识视频、评测视频、合集视频投稿前检查。B站更重视分区、标题、标签、简介和素材授权，尤其要避免搬运、盗用片段和音乐版权问题。

推荐素材横屏视频 1920x1080；竖屏视频 1080x1920；封面 1920x1080 或 1280x720。
重点检查分区错误、标题误导、标签不实、未授权音乐/视频片段、字幕来源、转载声明。
后台复核上传大小、时长、编码、帧率、HDR、互动视频、字幕和封面裁切。
请使用 $bid-bilibili-platform-rules，检查这个 B站投稿方案。
请输出：推荐分区、标题风险、标签建议、简介要补充的授权说明、封面风险、后台复核项。

快手：短视频、直播、电商

$bid-kuaishou-platform-rules

适合快手短视频、直播间预告、电商口播、商品卡片内容发布前检查。快手的直播和电商内容要更关注商品宣称、价格承诺、售后承诺和私下交易。

推荐素材竖屏视频 1080x1920；横屏视频 1920x1080；封面 1080x1920 或 1080x1440。
重点检查商品功效夸大、私下交易、二维码导流、直播间话术、虚假价格、健康/食品/美妆宣称。
后台复核直播封面、标题长度、视频大小与时长、电商类目规则、挂车和本地生活权限。
请使用 $bid-kuaishou-platform-rules，审查这段快手直播电商口播。
请重点看：商品宣称、价格承诺、售后承诺、私下交易、导流、敏感类目和必须删改话术。

微博：图文、视频、长文、话题

$bid-weibo-platform-rules

适合热点内容、品牌传播、图文长文、活动话题发布前检查。微博的风险经常来自话题、公共事件、隐私名誉、商业披露和未经核实的信息。

推荐素材单图 1080x1440、1080x1350 或 1080x1080；竖屏视频 1080x1920；横屏视频 1920x1080。
重点检查热点谣言、话题蹭热、隐私泄露、名誉侵权、未披露商业合作、截图和素材版权。
后台复核视频大小与时长、GIF 支持、长文编辑器、话题/超话规则、商业推广权限。
请使用 $bid-weibo-platform-rules，审查这条微博文案。
请重点看：话题是否匹配、是否有未经证实的信息、是否侵犯隐私名誉、是否需要商业合作披露。

更新与维护

平台规则会变，所以这个包的正确用法不是“一次写死永远不改”，而是把它当作规则底座。后续发现新规则时，优先更新对应平台的 references/*.md，再重新压缩打包。

本地更新流程

  1. 找到平台 reference

    例如抖音规则在 01-bid-douyin-platform-rules\bid-douyin-platform-rules\references\douyin.md。

  2. 补充新规则和来源

    把新规则写成“硬规则、建议规格、待后台复核”三类，不要把灰度功能当成稳定规则。

  3. 重新校验 Skill
    python "C:\Users\rjl20\.codex\skills\.system\skill-creator\scripts\quick_validate.py" "平台Skill文件夹路径"
  4. 重新压缩

    更新后重新生成平台独立 zip 和总 zip，确保客户拿到的是最新版本。

常见问题

1. 安装后 Codex 没有触发 Skill

先检查路径是否是 %USERPROFILE%\.codex\skills\bid-douyin-platform-rules\SKILL.md 这种结构。如果多套了一层资产目录，Codex 可能扫描不到。复制正确后重启客户端。

2. 为什么不直接写死所有平台上传大小和时长？

因为很多上传限制和账号权限、版本灰度有关，公开文档也不总是完整同步。这个包会给出推荐规格和公开来源，但把不稳定字段标成“需后台复核”，这是为了避免客户把经验值误当成官方承诺。

3. 这些 Skill 能保证内容一定审核通过吗？

不能。它们能帮助提前发现明显风险，减少返工和违规概率，但平台审核受账号状态、内容上下文、实时规则、投诉举报、人工复核等因素影响，不能承诺一定通过。

4. zip 文件怎么用？

zip 是方便分发的安装包。实际安装时解压后，把里面的同名 Skill 文件夹复制到 %USERPROFILE%\.codex\skills。如果你已经拿到未压缩文件夹，直接复制文件夹即可。

5. 抖音媒体检查脚本报错怎么办？

如果提示找不到 Python，先安装 Python。图片尺寸检查需要 Pillow，视频时长和编码检查需要 ffprobe。没有这些依赖时，仍可让 Codex 根据规则文本做人工审稿。

py -3 -m pip install pillow
winget install Gyan.FFmpeg

6. 如何给客户解释“后台复核项”？

可以这样说：平台上传限制会跟账号权限和产品版本有关，我们已经按公开规则和经验给出建议，但最终发布前仍需要以你账号后台当时显示的限制为准。
